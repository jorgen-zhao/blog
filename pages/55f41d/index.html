    物联网架构

    # 物联网层次划分

    # 行业应用

    职责：数据呈现以及客户交互 参照：数据分析IoTA

    # 云

    职责：解决数据如何存储、检索、使用、业务规划、安全、维护、费用等 参照： 设备接入IoTDA → Device Access 1. 设备连接：多网络、多协议、系列化SDK 2. 消息通信：物理模型、配置下发（命令、规则……）、订阅推送、设备影子（非实时在线 → 缓存类） 3. 设备管理：设备实时监控、OTA升级（FOTA、SOTA）、告警管理、设备远程诊断、设备批量操作、设备联动规则、分组标签、文件上传（设备物理描述profile） 4. 数据流转：数据转发规则

    # 管

    职责：接入和传输网络 参照：工业物联网关 → 主流物联网接入技术应用场景划分

    # 端

    职责：信息收集和信号处理 参照：LiteOS → 轻量级物联网操作系统
